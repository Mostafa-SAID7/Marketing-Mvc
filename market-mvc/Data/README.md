## Core Components

### AppDbContext

The main DbContext that orchestrates all entity configurations and manages database operations.

**Features:**
- Applies all entity configurations
- Implements audit field updates (CreatedAt, UpdatedAt, DeletedAt, etc.)
- Supports soft deletes
- Handles change tracking

```csharp
// Usage
var products = await context.Products.ToListAsync();
var newProduct = new Product { /* ... */ };
context.Products.Add(newProduct);
await context.SaveChangesAsync();
```

### Configurations

Individual configuration files for each entity following the fluent API pattern:

- **ProductConfiguration** - Decimal precision, indexes, relationships
- **OrderConfiguration** - Value objects (PersonName, Address), relationships
- **OrderItemConfiguration** - Composite keys, relationships
- **CategoryConfiguration** - Unique indexes, soft delete filters
- **CustomerConfiguration** - Value objects, unique indexes
- **ItemConfiguration** - Decimal precision, unique indexes
- **GlobalEntityConfiguration** - Soft delete filters for all entities

### Migrations

Database migration files generated by Entity Framework Core:

- `Migrations/` - Migration history
- Each migration includes Up/Down methods

**Apply migrations:**
```bash
dotnet ef database update
```

### Seeds

See `Seeds/` directory for database seeding documentation.

The new seeding structure uses:
- `CategorySeeder` - Seeds product categories
- `CustomerSeeder` - Seeds customers
- `ProductSeeder` - Seeds products
- `OrderSeeder` - Seeds orders

**Execute seeders:**
```csharp
var orchestrator = app.Services.GetRequiredService<SeederOrchestrator>();
await orchestrator.ExecuteAsync(context);
```

### Unit of Work Pattern

Coordinates multiple repositories and manages transactions:

```csharp
public interface IUnitOfWork
{
    IOrderRepository OrderRepository { get; }
    IProductRepository ProductRepository { get; }
    Task SaveAsync();
}
```

**Usage:**
```csharp
var order = new Order { /* ... */ };
unitOfWork.OrderRepository.Add(order);
await unitOfWork.SaveAsync();
```

## Configuration System

### Entity Configuration Example

Each entity has a configuration implementing `IEntityTypeConfiguration<T>`:

```csharp
public class ProductConfiguration : IEntityTypeConfiguration<Product>
{
    public void Configure(EntityTypeBuilder<Product> entity)
    {
        // Property configurations
        entity.Property(p => p.Price).HasPrecision(18, 2);
        
        // Indexes
        entity.HasIndex(p => p.Sku).IsUnique();
        
        // Relationships
        entity.HasOne(p => p.Category)
            .WithMany(c => c.Products)
            .OnDelete(DeleteBehavior.SetNull);
    }
}
```

### Application in AppDbContext

```csharp
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    base.OnModelCreating(modelBuilder);
    
    // Apply global configurations
    modelBuilder.ApplyGlobalConfigurations();
    
    // Apply entity configurations
    modelBuilder.ApplyConfiguration(new ProductConfiguration());
    modelBuilder.ApplyConfiguration(new OrderConfiguration());
    // ... etc
}
```

## Key Features

### Soft Delete Support

All entities inheriting from `BaseEntity` automatically support soft deletes:

```csharp
// Delete marks record as deleted, doesn't remove from database
await context.Products.FirstAsync().Delete();

// Query automatically filters deleted records
var products = await context.Products.ToListAsync(); // Only non-deleted

// Include deleted records
var allProducts = await context.Products
    .IgnoreQueryFilters()
    .ToListAsync();
```

### Value Objects

Complex properties are modeled as owned entities:

```csharp
// Order has PersonName and Address value objects
order.CustomerName = new PersonName("John", "Doe");
order.ShippingAddress = new Address("123 Main St", "NYC", ...);

// Automatically mapped to database columns
// CustomerFirstName, CustomerLastName, ShippingStreet, etc.
```

### Audit Trails

Automatic audit field updates on save:

- `CreatedAt` - When entity was created
- `CreatedBy` - User who created it
- `UpdatedAt` - Last update timestamp
- `UpdatedBy` - User who last updated
- `DeletedAt` - When soft deleted
- `DeletedBy` - User who deleted

### Indexes

Strategic indexes for query performance:

```csharp
// Unique indexes
entity.HasIndex(c => c.Email).IsUnique();

// Composite indexes
entity.HasIndex(oi => new { oi.OrderId, oi.ProductId });

// Performance indexes
entity.HasIndex(p => p.CreatedAt);
entity.HasIndex(p => p.Status);
```

## Common Tasks

### Add a New Entity

1. Create entity class in `Models/entity/`
2. Create configuration in `Data/Configurations/`
3. Add DbSet to `AppDbContext`
4. Create migration: `dotnet ef migrations add AddYourEntity`
5. Apply migration: `dotnet ef database update`

### Query Data

```csharp
// Simple query
var products = await context.Products.ToListAsync();

// With filtering
var activeProducts = await context.Products
    .Where(p => p.IsActive)
    .ToListAsync();

// With relationships
var orders = await context.Orders
    .Include(o => o.Customer)
    .Include(o => o.OrderItems)
    .ToListAsync();

// Include deleted
var deletedProducts = await context.Products
    .IgnoreQueryFilters()
    .Where(p => p.IsDeleted)
    .ToListAsync();
```

### Modify Data

```csharp
// Add
var product = new Product { /* ... */ };
context.Products.Add(product);
await context.SaveChangesAsync();

// Update
var product = await context.Products.FindAsync(id);
product.Name = "New Name";
await context.SaveChangesAsync();

// Delete (soft delete)
var product = await context.Products.FindAsync(id);
context.Products.Remove(product);
await context.SaveChangesAsync();

// Hard delete
var product = await context.Products.IgnoreQueryFilters().FindAsync(id);
context.Products.Remove(product);
await context.SaveChangesAsync();
```

## Database Migrations

### Create Migration

```bash
dotnet ef migrations add {DescriptiveName}
```

Example:
```bash
dotnet ef migrations add AddProductDiscountPrice
```

### Apply Migrations

```bash
# Update to latest
dotnet ef database update

# Update to specific migration
dotnet ef database update AddProductDiscountPrice

# Revert last migration
dotnet ef database update {PreviousMigration}
```

### Remove Migration

```bash
dotnet ef migrations remove
```

### List Migrations

```bash
dotnet ef migrations list
```

## Performance Optimization

### Query Performance

```csharp
// Use Select for specific fields
var productNames = await context.Products
    .Select(p => new { p.Id, p.Name })
    .ToListAsync();

// Use AsNoTracking for read-only queries
var products = await context.Products
    .AsNoTracking()
    .ToListAsync();

// Use pagination
var page = await context.Products
    .Skip((pageNumber - 1) * pageSize)
    .Take(pageSize)
    .ToListAsync();
```

### Index Strategy

Indexes are created on:
- Frequently filtered columns
- Frequently sorted columns
- Foreign key columns
- Unique constraints

## Troubleshooting

### Issue: "An instance of entity type cannot be tracked"

**Cause**: Entity loaded in a different DbContext instance
**Solution**: Use `AsNoTracking()` for read-only queries or reload in current context

### Issue: "The instance of entity type ... cannot be tracked because another instance with the same key value"

**Cause**: Multiple instances with same ID
**Solution**: Use `LocalQueryExecutor` or reload entity

### Issue: Migrations not applying

**Cause**: Connection string, permissions, or migrations pending
**Solution**: 
```bash
dotnet ef database update --verbose
```

## Best Practices

✅ **DO:**
- Use `async/await` for database operations
- Implement Unit of Work pattern
- Use value objects for complex types
- Create indexes on frequently queried columns
- Use soft delete for audit trails

❌ **DON'T:**
- Execute N+1 queries
- Load unnecessary related entities
- Modify entities outside DbContext
- Skip migrations in production
- Hardcode connection strings
