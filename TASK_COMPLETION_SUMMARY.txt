HOW TO USE
================================================================================

1. SEEDING DATA:
   - Seeders auto-run on application startup
   - Located in Data/Seeds/
   - Individual files: CategorySeeder, CustomerSeeder, ProductSeeder, OrderSeeder
   - Orchestrator manages execution order

2. DATABASE CONFIGURATION:
   - Entity configurations in Data/Configurations/
   - One configuration per entity
   - Easy to find and modify
   - Implements IEntityTypeConfiguration pattern

3. ADDING NEW ENTITY:
   Step 1: Create entity in Models/entity/
   Step 2: Create configuration in Data/Configurations/
   Step 3: Add DbSet to AppDbContext
   Step 4: Create seeder in Data/Seeds/ (if needed)
   Step 5: Create migration: dotnet ef migrations add AddYourEntity
